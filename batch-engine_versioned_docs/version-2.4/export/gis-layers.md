# GIS Layers

These exports are GIS layers, written in the project's configured format
(see [Output folder and formats](./#output-folder-and-formats); `.xxx` denotes
the chosen extension). The
attributes of each layer are defined in the *GIS Layer Attributes* appendix
(Infrastructure / Architecture / Cable System / Conduit System layers).

## Network Nodes

`Net_Nodes.xxx` — a punctual layer with all nodes of the optimized network:
nodes loaded from a source, created manually, and auto-generated by Setics Sttar.
*(Architecture Layers.)*

## Network Service Areas

Polygon layers representing the area served by each node of `Net_Nodes`. The two
variants differ in polygon rendering:

- **Convex Hulls** — `Net_ServiceAreas.xxx`: simple single-part polygons with no
  holes; more aesthetic but areas can overlap.
- **Voronoi Diagram** — `ResOpt_ServiceAreasVoronoi.xxx`: non-overlapping, but
  polygons can be complex (several parts and/or holes).

*(Architecture Layers.)*

## Network Endpoints

`Net_Endpoints.xxx` — a punctual layer with all endpoints of the optimized
network (loaded from a source or created by hand). *(Architecture Layers.)*

## Network Lines

`Net_Lines.xxx` — the architecture of the design; node-to-node and
node-to-endpoint lines follow the infrastructure. *(Architecture Layers.)*

## Network Pathways

`Net_Pathways.xxx` — the network's actual layout: all infrastructure sections
(arteries) used by the network cables. Feeder and distribution sections can be
exported separately. *(Infrastructure Layers.)*

## Pathway Access Structures

`Net_PathwayAccessStructures.xxx` — all punctual infrastructures actually used by
the network: poles and chambers from the used pathways, plus the served-outlet
and used-cabinet locations. *(Infrastructure Layers.)*

## Available Pathways and Connections between Supports

`Infra_PotentialPathways.xxx` — all reusable pathways explored by Setics Sttar
(loaded, hand-created, and auto-generated linear infrastructures). Useful to
analyze the software's decisions. *(Infrastructure Layers.)*

## Available Pathway Access Structures

`Infra_PotentialPathwayStructures.xxx` — all reusable pathway access structures
explored (poles, chambers, potential chambers — the extremities of the linear
infrastructures, excluding outlets and cabinets). *(Infrastructure Layers.)*

## Cables

Selecting **Cables** exports three layers:

- `Net_Cables.xxx` — the linear routes of the optical cables.
- `Net_Cables_ClosureSections.xxx` — cable sections from one splice closure to
  another (same route as `Net_Cables`, cut at the closures).
- `Net_Cables_PathwaySections.xxx` — all cable sections (same route, cut at each
  intersection).

*(Cable System Layers.)*

## Network Splice Closures

`Net_SpliceClosures.xxx` — a punctual layer giving the location and type of every
splice closure in the network. *(Cable System Layers.)*

## Cable Devices

`Net_CableDevices.xxx` — a punctual layer giving the location and type of every
cable device in the network. *(Cable System Layers.)*

## Ducts

`Net_Ducts.xxx` and `Net_DuctsSections.xxx` — polyline layers representing the
ducts and duct sections used in the design. *(Conduit System Layers.)*

## Plugins

Each additionally installed plugin adds a line to the **Plugins** box, activated
or deactivated with its checkbox.
